Overview

Welcome to the Task Management API documentation. This comprehensive guide will help you integrate with our powerful REST API for project and task management.

API Base URL
http://localhost:5120/api

What is the Task Management API?

The Task Management API is a modern, production-ready REST API built with .NET 8 and ASP.NET Core following clean architecture principles. It provides comprehensive endpoints for managing projects, tasks, team members, real-time notifications, and user authentication.

Key Features

  • Project Management: Create, update, and manage projects with team members
  • Task Management: Organize tasks with priorities, dependencies, and time tracking
  • Real-time Updates: SignalR integration for live notifications and updates
  • JWT Authentication: Secure token-based authentication and authorization
  • Role-based Access: Fine-grained permission control with multiple roles
  • Soft Delete: Safe data deletion with restoration capabilities
  • Audit Logging: Complete audit trail for compliance and tracking
  • Caching: Redis-backed caching for optimal performance
  • Logging: Comprehensive structured logging with Serilog
  • Performance Monitoring: Real-time performance metrics
  • Error Handling: Consistent error responses with detailed messages
  • Pagination: Efficient data retrieval with pagination support
  • Docker Support: Ready for containerized deployment

Quick Start

Get up and running with the Task Management API in minutes.

1. Authentication

First, register a new user or login to get your JWT token:

POST /api/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "SecurePass123!",
  "firstName": "John",
  "lastName": "Doe"
}

2. Create a Project

Create your first project:

POST /api/projects
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "My First Project",
  "description": "A project for managing tasks"
}

3. Create a Task

Add tasks to your project:

POST /api/tasks
Authorization: Bearer {token}
Content-Type: application/json

{
  "projectId": "project-id",
  "title": "Implement feature",
  "description": "Build the new feature",
  "priority": "High",
  "dueDate": "2026-04-15"
}

Authentication

The Task Management API uses JWT (JSON Web Tokens) for authentication. All requests to protected endpoints must include a valid JWT token in the Authorization header.

Getting a Token

Send your credentials to the login endpoint:

POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "SecurePass123!"
}

Using the Token

Include the token in the Authorization header for all subsequent requests:

GET /api/projects
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Token Expiration

Tokens expire after 1 hour. When a token expires, you'll receive a 401 Unauthorized response. Simply login again to get a new token.
