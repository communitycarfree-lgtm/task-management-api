🚀 Core Capabilities

A full-featured engine for professional task and project management at any scale.

Task Dependency Graph

Model real-world workflows where task B cannot start until task A is complete. Circular dependencies are automatically detected and rejected.

  • Blocking & blocked-by links
  • Cycle detection on write
  • Cascade status propagation

Time Tracking

Precise per-task time logging with estimated vs. actual hours — giving project managers accurate capacity data for planning.

  • Start / stop time entries
  • Estimated vs. actual delta
  • Per-user & per-project rollup

Status State Machine

Tasks move through a defined lifecycle — Todo → InProgress → InReview → Done. Each transition is validated server-side; invalid jumps return a clear error.

  • Validated transitions only
  • Backward re-open allowed
  • Transition timestamp recorded

Filtering & Pagination

Every list endpoint supports rich server-side filtering — by status, priority, assignee, date range — with offset-based pagination and configurable page sizes.

  • Multi-field filter params
  • Configurable page size
  • Consistent sort ordering

🗂️ Data Integrity & Audit Trail

Every write leaves a complete, tamper-evident record. Nothing is ever silently lost.

Full Audit Trail

Every entity in the system carries three actor fields stamped automatically on each write — no controller code required.

  • CreatedBy — JWT subject on first insert
  • UpdatedBy — JWT subject on every update
  • DeletedBy — JWT subject on soft-delete

Soft Delete & Recovery

No data is immediately destroyed on DELETE. Records are archived with a timestamp and actor, then filtered from all GET responses automatically.

  • IsDeleted boolean flag
  • DeletedAt UTC timestamp
  • EF Core global query filter applied

Structured Audit Logging

Every state-changing database operation emits a dedicated Serilog AUDIT event with entity type, action, actor, and timestamp — routable to any sink (file, Seq, Elastic).

  • Entity name & primary key logged
  • Action: Insert / Update / SoftDelete
  • Correlated with HTTP RequestId

Rate Limiting

A sliding-window rate limiter enforces fair usage across all clients. Violations return a clear 429 response with a Retry-After header.

  • 100 requests / minute per account
  • X-RateLimit-Remaining header
  • Configurable per environment
How actor stamping works
ICurrentUserService resolves the actor ID from the JWT sub claim and passes it into BaseDbContext.SaveChangesAsync(). If no authenticated user is present (anonymous request), the actor is recorded as system. This happens automatically for every module — Projects, Tasks, and Notifications — with zero per-controller code.

⚠️ Error Handling

Consistent, structured error responses across every endpoint make integration straightforward.

FluentValidation

All incoming requests are validated before reaching the service layer. Validation errors are aggregated into a single 400 response with per-field detail.

Business Rule Enforcement

Domain violations (circular dependencies, invalid status transitions) return 409 Conflict or 422 Unprocessable Entity with a human-readable detail field.

Trace IDs on Every Error

Every error response includes the ASP.NET Core traceId (tied to the Serilog RequestId). Support teams can find the full request log in seconds.

Consistent Error Envelope

All errors use the RFC 7807 Problem Details format — status, title, detail, traceId, and an optional errors map — no surprises for clients.
