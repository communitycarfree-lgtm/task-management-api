Architecture Overview

Understanding the design and inner workings of the Task Management API

The Task Management API is built using Clean Architecture principles. This ensures the business logic remains independent of frameworks, UI, and external databases, making the system highly testable and maintainable.

Architectural Goal

Our primary goal is to provide a scalable foundation that can support thousands of concurrent users while maintaining strict data integrity and performant response times.

Layered Design

The solution is organized into four distinct layers, following the Onion Architecture pattern:

Presentation Layer

The API project itself. Handles HTTP requests, authentication, and global error handling.

  • Controllers & Minimal APIs
  • Middleware Components
  • Swagger Documentation
  • SignalR Hubs

Application Layer

Contains all business use cases, MediatR handlers, and request validation logic.

  • Commands & Queries
  • DTOs & Mappings
  • Fluent Validators
  • Domain Event Handlers

Infrastructure Layer

Implements data access, external service integrations, and cross-cutting concerns.

  • EF Core Context
  • Repositories & Unit of Work
  • Redis Caching
  • Email/SMS Services

Domain Layer

The core of the system. Contains entities, value objects, and domain exceptions.

  • Entities
  • Domain Events
  • Value Objects
  • Core Interfaces

Dependency Direction

Presentation Layer (API)
            ↓
Application Layer (Logic)
            ↓
Infrastructure Layer (Data)
            ↑
  Domain Layer (Entities)

Design Patterns

CQRS Pattern

We use Command Query Responsibility Segregation to separate read and write operations, allowing for optimized data models and better scaling.

  • Commands: Change system state (e.g., CreateTask, AssignMember).
  • Queries: Retrieve data without side effects (e.g., GetProjectDetails).

Repository & Unit of Work

Abstracts data access and ensures multiple operations are treated as a single atomic transaction.

Strategy Pattern

Used for flexible notification delivery and dynamic priority calculations.

Technology Stack

.NET 8/9

Core framework for high-performance API development.

EF Core

Modern ORM for database operations with SQL Server.

Redis

Distributed caching for high-speed data retrieval.

SignalR

Real-time bidirectional communication for notifications.
