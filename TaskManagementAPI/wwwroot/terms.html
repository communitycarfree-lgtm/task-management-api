To ensure high availability for all users, standard accounts are limited to 100 requests per minute. Exceeding this threshold results in 429 Too Many Requests responses.

All API communication must occur over HTTPS. Plain HTTP requests will be rejected. Authentication tokens must be handled using secure client-side storage and must never be embedded in publicly accessible code.
