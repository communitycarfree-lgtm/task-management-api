Projects

Manage project workspaces, settings, and team membership.

Project Management
GET /api/projects
List all projects with pagination and filtering options.
POST /api/projects
Create a new project workspace with optional description.
GET /api/projects/{id}
Retrieve detailed information about a specific project.
PUT /api/projects/{id}
Update project metadata, name, and settings.
DELETE /api/projects/{id}
Soft-delete (archive) a project. Data is retained.
Team Collaboration
GET /api/projects/{id}/members
List all members of a project with their assigned roles.
POST /api/projects/{id}/members
Invite a user to the project with an assigned role.
PUT /api/projects/{id}/members/{userId}
Change a member's role within the project.
DELETE /api/projects/{id}/members/{userId}
Revoke a member's access to the project.

Tasks

Fine-grained control over task creation, status transitions, and assignment.

Task Operations
GET /api/tasks
Query tasks across all projects with filtering and sorting.
POST /api/tasks
Create a new task within a project context.
PUT /api/tasks/{id}
Update task title, description, priority, and due date.
DELETE /api/tasks/{id}
Remove a task permanently from the system.
PUT /api/tasks/{id}/status
Transition task through its lifecycle states.
PUT /api/tasks/{id}/assignee
Assign or reassign task ownership to a team member.
Task Status Values
Valid transitions: Todo → InProgress → InReview → Done. Backward transitions are allowed for re-opening tasks.
Dependencies
GET /api/tasks/{id}/dependencies
List all blocking and blocked-by dependencies for a task.
POST /api/tasks/{id}/dependencies
Add a dependency link. Circular dependency detection runs automatically.
DELETE /api/tasks/{id}/dependencies/{depId}
Remove a specific dependency link between two tasks.

Time Tracking

Log, review, and manage time entries against individual tasks for accurate capacity planning.

Time Entries
GET /api/tasks/{id}/time-entries
Retrieve all time entries for a task, including estimated vs. actual breakdown.
POST /api/tasks/{id}/time-entries
Log time worked against a task with an optional note and duration.
PUT /api/time-entries/{id}
Correct a time entry's duration, date, or description.
DELETE /api/time-entries/{id}
Permanently remove a time entry from the task log.

Log Time — Request Body

JSON 
{
  "durationMinutes": 90,
  "loggedAt": "2026-06-20T14:00:00Z",
  "note": "Completed OAuth token validation logic"
}
Estimated vs. Actual
Set estimatedMinutes on a task at creation. The API automatically computes variance between estimate and logged actuals on each GET /api/tasks/{id}/time-entries response.

Users

Manage user profiles, credentials, and administrative roles.

Profile Management
GET /api/users/{id}
Retrieve a user profile including role assignments.
PUT /api/users/{id}
Update a user's display name and profile details.
PUT /api/users/{id}/password
Securely change the account password with old-password verification.
Administration
GET /api/users
List all registered platform users. Requires Admin role.
POST /api/users/{id}/roles

Notifications

Manage your personal notification feed. Real-time delivery via SignalR.

Alert Inbox
GET /api/notifications
Retrieve paginated alerts for the currently authenticated user.
PUT /api/notifications/{id}/read
Mark a specific notification as read.
DELETE /api/notifications/{id}
Permanently delete a notification from the inbox.

Authentication

JWT Bearer token issuance and refresh. All tokens are signed with HMAC-SHA256.

Identity Provider
POST /api/auth/register
Create a new platform identity and receive an access token pair.
POST /api/auth/login
Exchange credentials for a JWT access + refresh token pair.
POST /api/auth/refresh
Issue a new access token using a valid, non-expired refresh token.

Response Codes

Every response follows standard HTTP semantics. Handle these codes in your integration for a robust client.

Success Codes
200 OK
Request succeeded. Response body contains the requested resource or operation result.
201 Created
Resource was created. Location header contains the URL of the new resource.
204 No Content
Operation succeeded with no response body. Typical for DELETE and some PUT endpoints.
Client Error Codes
400 Bad Request
Request body failed validation. Response includes a errors array detailing each field violation.
401 Unauthorized
Missing or expired JWT token. Re-authenticate via POST /api/auth/refresh.
403 Forbidden
Token is valid but your role does not permit this action. Check the endpoint's required role.
404 Not Found
Resource does not exist or has been soft-deleted. Soft-deleted resources are not returned in GET responses.
409 Conflict
State conflict — e.g. a circular task dependency was detected, or a duplicate resource key exists.
422 Unprocessable Entity
Request was well-formed but violates business rules — e.g. an invalid task status transition.
429 Too Many Requests
Rate limit exceeded (100 req/min per account). Check the Retry-After response header for the reset window.
Server Error Codes
500 Internal Server Error
Unexpected server fault. A structured error ID is returned — include it when filing a support request.
503 Service Unavailable
Server is temporarily down for maintenance or overloaded. Retry with exponential back-off.

{
  "status": 422,
